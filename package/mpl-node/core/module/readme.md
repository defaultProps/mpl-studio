# 模板组件


## 类别

社区版、企业版功能。

有三种模板组件：业务模板、自定义模板、系统模板。


- 业务模板：通过平台拖拉拽生成的模板。可一键同步更新。
- 系统模板：mpl系统内置的模板。系统维护版本，自动升级最新版本且兼容旧版本
- 自定义模板：挂载预览平台内置的模板【通常时全局挂载使用的组件】。

都不存在版本概念或模糊了版本自动升级最新组件。


业务模板是用户手动维护的业务组件集合。仅在项目级别使用。

系统模板是系统官方维护的模板，全平台使用。

自定义模板是开发手动维护的vue组件，仅仅在项目级别使用。


## 使用方式

模板提供以下配置项。
- 入参变量【变量名称、数据类型】
- 入参方法【方法名称、参数】
- 出参方法【方法参数、返回值】
- 出参变量【变量名称、数据类型】
- 模板名称、模板描述信息、更新日期等其他辅助信息

调用都是通过模板实现。


## 系统模板

1、只读不可更改。固定UI、排版、主题、功能、交互等细节。
2、企业版或社区版。
3、使用方式
```shell
  npm i @mpl/core -S
  import mpl from '@mpl/core'
  app.use(mpl)

  demo.vue
  <mpl-module-excel-table v-model="tableData" />
```

## 业务模板。

1、仅限制在mpl平台拖拉拽开发
2、企业版。
3、开发后自动推送代码【推送到固定的文件夹路径、自动配置路由挂载全局require.context()】

## 自定义。

1、挂载预览平台手动开发
2、企业版。
3、固定文件夹下开发。require.context() 自动挂载全局。
